# 🚀 簡易ToDoアプリ 開発ガイド

このプロジェクトは、プログラミング初心者がWeb開発の基本（**フロントエンド、バックエンド、データベース連携**）を学ぶための雛形です。

## 🛠️ 技術スタック

* **フロントエンド:** HTML, CSS, Jinja2 (テンプレートエンジン)
* **バックエンド:** Python, Flask
* **データ:** SQLite (Flask-SQLAlchemy経由で使用)

## 1. 🖥️ フロントエンドの担当範囲

フロントエンドは、ユーザーがブラウザで直接操作する**見た目と動き**を担当します。

### 📍 記述する場所

| ファイル | 内容 | 記述方法 |
| :--- | :--- | :--- |
| `templates/index.html` | **ページの構造と表示** | **HTML**でタスク一覧やフォームの配置を定義し、**Jinja2**（`{% for todo in todos %}` や `{{ todo.content }}`）を使ってバックエンドから渡されたデータを動的に表示します。 |
| `static/css/style.css` | **デザインと見た目** | **CSS**で文字の色、レイアウト、ボタンのデザインなどを設定します。 |
| `static/js/main.js` | **動きと対話** | **JavaScript**で、ボタンクリック時の動作や、ページをリロードせずにデータを変更する処理などを記述します。（現在は空のファイルです） |

### 💡 初学者のポイント

* 見た目を変更したいときは、**`index.html`**と**`style.css`**を編集します。
* Pythonから渡された変数を表示する際は、必ず二重波括弧 **`{{ variable }}`** を使ってください。

---

## 2. 🧠 バックエンドの担当範囲

バックエンドは、**データの永続的な保存、データベースとの連携、リクエストの処理**など、Webアプリの頭脳となる部分を担当します。

### 📍 記述する場所

メインのコードはすべて **`app.py`** に **Pythonコード**として記述します。

| バックエンドの役割 | `app.py`での実装内容 |
| :--- | :--- |
| **① データの構造定義** | **`class Todo(db.Model):`** のように、データベースに保存するデータの「設計図」（テーブル）を定義します（SQLAlchemyを使用）。 |
| **② URLルーティング** | **`@app.route('/')`** のように、ユーザーからのアクセスURLに対して実行する処理（関数）を定義します。 |
| **③ データベース操作** | フォームのデータを受け取り、DBに保存（**Create**）したり、DBからデータを読み出してフロントエンドに渡したり（**Read**）するロジックを記述します。 |

### 💡 初学者のポイント

* ユーザーが新しいタスクを**追加**したい場合の処理は、`app.py`内の **`if request.method == 'POST':`** ブロックに追加します。
* データベース操作はSQL文ではなく、Pythonのコード（例：`db.session.add(new_todo)`）で行うため、直感的に開発を進められます。

---

## 3. 🔗 フロントとバックエンドの連携フロー

ToDoタスクを**追加**する際の基本的な流れは以下の通りです。

1.  **[フロント]** ユーザーがフォームに入力し、ボタンをクリックする。
2.  **[バックエンド]** Flask (`app.py`) がそのフォームデータを **POSTリクエスト**として受け取る。
3.  **[バックエンド]** `app.py`がデータを検証し、**SQLAlchemy**を使って`instance/todo.db`に新しいTodoデータを保存する。
4.  **[バックエンド]** 処理完了後、`app.py`はデータベースから最新のTodoリストをすべて取得する。
5.  **[バックエンド]** 取得したリストを **Jinja2**（HTMLテンプレート）に渡してレンダリングする。
6.  **[フロント]** ブラウザに**最新のタスクリスト**が返され、表示される。
