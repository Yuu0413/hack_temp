2. 🖥️ フロントエンドの担当範囲
フロントエンドは、ユーザーがブラウザで直接操作する見た目と動きを担当します。
📍 記述する場所
templates/index.html:

HTMLでタスク一覧やフォームの配置といったページの構造を定義します。
バックエンドから渡されたデータ（例：todos）を表示するために、Jinja2の構文（{% for todo in todos %} や {{ todo.content }}）を使用します。

static/css/style.css:

CSSで文字の色、レイアウト、ボタンのデザインといった見た目を設定します。

static/js/main.js:

JavaScriptで、ボタンクリック時の動作や、ページをリロードせずにデータを変更する処理などを記述します。（現在は空のファイルです）

💡 初学者のポイント

見た目を変更したいときは、**index.htmlとstyle.css**を編集します。
Pythonから渡された変数を表示する際は、必ず二重波括弧 {{ variable }} を使ってください。

3. 🧠 バックエンドの担当範囲
バックエンドは、データの永続的な保存、データベースとの連携、リクエストの処理など、Webアプリの頭脳となる部分を担当します。
📍 記述する場所
メインのコードはすべて app.py に Pythonコードとして記述します。
バックエンドの役割
バックエンドの役割app.pyでの実装内容① データの構造定義class Todo(db.Model): のように、データベースに保存するデータの「設計図」（テーブル）を定義します（SQLAlchemyを使用）。② URLルーティング@app.route('/') のように、ユーザーからのアクセスURLに対して実行する処理（関数）を定義します。③ データベース操作フォームのデータを受け取り、DBに保存（Create）したり、DBからデータを読み出してフロントエンドに渡したり（Read）するロジックを記述します。
💡 初学者のポイント

ユーザーが新しいタスクを追加したい場合の処理は、app.py内の if request.method == 'POST': ブロックに追加します。
データベース操作はSQL文ではなく、Pythonのコード（例：db.session.add(new_todo)）で行うため、直感的に開発を進められます。

4. 🔗 フロントとバックエンドの連携フロー
ToDoタスクを追加する際の基本的な流れは以下の通りです。

[フロント] ユーザーがフォームに入力し、ボタンをクリックする。
[バックエンド] Flask (app.py) がそのフォームデータを POSTリクエストとして受け取る。
[バックエンド] app.pyがデータを検証し、SQLAlchemyを使ってinstance/todo.dbに新しいTodoデータを保存する。
[バックエンド] 処理完了後、app.pyはデータベースから最新のTodoリストをすべて取得する。
[バックエンド] 取得したリストを Jinja2（HTMLテンプレート）に渡してレンダリングする。
[フロント] ブラウザに最新のタスクリストが返され、表示される。
